# MLTrainer — обучение Core ML моделей для FoodStory

Здесь живут инструменты разработчика для обучения моделей. **Это не часть
приложения** — код отсюда не попадает в App Store, он только готовит `.mlmodel`,
который потом лежит в `FoodStory/ML/` и компилируется Xcode при сборке.

## Классификатор категории блюда

`TrainCategoryModel.swift` обучает модель, которая по тексту рецепта (название +
ингредиенты) определяет категорию (`breakfast`, `soup`, `main`, `salad`,
`dessert`, `baking`, `drink`, `other`).

### Как переобучить

Из корня проекта на Mac:

```sh
xcrun --sdk macosx swift MLTrainer/TrainCategoryModel.swift
```

Результат: `FoodStory/ML/RecipeCategoryClassifier.mlmodel`. Дальше просто собери
приложение — Xcode сам скомпилирует модель.

### Как улучшить модель

Модель настолько умная, насколько разнообразны данные. Дописывай новые примеры в
массив `samples` (побольше на каждую категорию, с разными словами) и запускай
скрипт снова.

## Куда развивать дальше (для App Store)

- **Больше данных** — 64 примера хватает для демонстрации, но для магазина стоит
  собрать сотни (можно в CSV и грузить через `MLDataTable(contentsOf:)`).
- **Обучение на устройстве (MLUpdateTask).** Сейчас модель обучается один раз на
  Mac. Чтобы она *дообучалась у каждого пользователя на телефоне*, нужно сделать
  модель «updatable»: это делается через `coremltools` (Python) — модель
  помечается обновляемой, а в приложении её дообучает `MLUpdateTask`. Это заметно
  сложнее и делается отдельным шагом, когда базовая версия уже в сторе.
- **Персонализация уже работает** через `TasteModel` (обучается на устройстве на
  избранном и готовке) — это дополняет общий Core ML «мозг».
